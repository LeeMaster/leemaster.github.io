Thompson 正则表达式搜索算法

Posted on 2020-03-16 | In Compiler

用Rust 实现了 Regular Expression Search Algorithm 这份论文中提到的搜索算法

这份1968年的论文，现在看起来有点老了，但是论文写的很OK的，主要介绍了一个方法，通过将模式编译成为IBM 7704 机器代码来将一个文本中全部符合模式的字符全部识别出来。

论文的大概

论文中设计了一个程序，用来将一个正则表达式转换为一段IBM 7094汇编代码，从而可以接受任意的输入文本，并将匹配当前pattern的全部字符串输出出来。那么论文既然是实现了一个编译器用来将正则表达式生成为一段汇编识别自动机器，那么重要的工作就是设计这个编译器的算法。

编译器设计

这个编译器包含了三个并行的阶段。

  • 第一阶段语法过滤，这个阶段允许符合正则表达式的语法规范的句子通过，同时插入.符号显式标记正则表达式的两部分的连接
  • 第二个阶段将正则表达式转换为RPN模式，e.g. a|b -> ab|，即转换为逆波兰表达法（利用的是树IR表示）
  • 第三个阶段是生成汇编语言的阶段

逆波兰表达式

对于一个正则表达式a(b|c)*d 转换成为逆波兰表达式后 abc|*.d. 这里的.表达的是连接的意思，简单来说就是将两个链表直接连接到一起。

生成逆波兰表达式的算法如下描述：

  1. 申请两个栈expr和operator其中expr保存的是最后输出的表达式，operator临时保存运算符号。
  2. 从左到右读取字符
    1. 如果当前的字符x是操作数的话，直接压入栈expr中 -> if x in [a-z] then push x to expr
    2. 如果当前的字符x是(的话，将符号压入operator中 -> if x == '(' then push x to operator
    3. 如果当前的字符x是)的话，将operator中的符号依次弹出并压入expr中丢弃最后的( -> if x == ')' then loop push operator.pop() to expr
    4. 如果当前的字符x是)和(之外的运算符的话
      1. 如果operator栈顶的运算符为(的话直接将x压入operator
      2. 如果operator栈顶的运算符不为(的话，依次弹出operator压入expr，直到栈顶的运算符优先级高于x或者为(再将x压入operator，如果operator为空直接压入expr

转换RPN的关键就是用两个栈实现了优先级分析，因为RPN表达式自带优先级，通常对于一个运算表达式，完全可以表达成为一颗树，如果是中缀表达式的话，只需要进行一次中序遍历带计算就可以拿到结果，而在深处的表达式子树拥有更高的优先级。

Rust实现一下这个算法

编译算法的第三个阶段

经过前面两个阶段的处理，我们得到了一个逆波兰表达法的正则表达式，第三个阶段的核心是一个下推栈，从左到右处理逆波兰表达式：

  1. 当扫描到的字符为普通字符（[a-z]）直接生成一个单字符表达式推入栈中（论文中被定义为NNODE）
  2. 当扫描到的字符为二元运算符号（|和.）的时候，从栈中出两个表达式，组合后再推回去（论文中定义为CNODE）
  3. 当扫描到的字符为一元运算符号 (?+*)的时候，把栈顶的表达式出战栈，通过包装后生成新的表达式再推回去 （论文中定义为CNODE）
  4. 所有的字符处理完毕，栈中应该只有一个表达式了

第三个阶段其实很像我们从叶子节点构建一棵树的过程，不停地移入规约最后完成一棵树的样子，不过论文中比较脑袋疼的地方是IBM 7094的汇编指令，还好找到了汇编指令：

7094的汇编指令

首先我们看一下7094机器的组成

  • 7094的地址范围是，0-32768（15位地址），按照字寻址，这个字长是36位。
  • 38位的累加器，35位数字大小，剩余三位，一位是符号位，一位是用来表示无符号字，还有一个是进位
  • 七个15位的索引寄存器 XR，通常包含一个字编址的指针，寄存器的范围从1到7
  • 15位的指令寄存器 IC

讨论了组成以后，那就看一下指令格式吧

从高地址到低地址，分别为address,index,decrement,op,一般来说，decrement这个字段没啥用，一般用来放第二个操作数，比如TRA label, index

详细的指令参考 IBM 7094 sheet
